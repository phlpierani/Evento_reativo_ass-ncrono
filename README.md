# 🎟️ CodeChella - API Reativa com Spring WebFlux

## 📌 Sobre o projeto

O **CodeChella** é uma API desenvolvida para uma plataforma de **venda de ingressos para eventos**, projetada para suportar alto volume de requisições e usuários de diferentes países.

A aplicação foi construída utilizando **programação reativa**, permitindo **processamento assíncrono e não bloqueante**, garantindo melhor desempenho em cenários de alta concorrência.

Este projeto foi desenvolvido durante um curso da plataforma Alura, com foco em **Spring WebFlux e arquitetura reativa**.

---

# 🚀 Tecnologias utilizadas

- Java
- Spring Boot
- Spring WebFlux
- Reactor (Mono e Flux)
- Netty
- PostgreSQL
- R2DBC
- H2 Database
- WebClient
- WebTestClient
- Apache AB
- DeepL API

---

# ⚙️ Conceitos aplicados

Durante o desenvolvimento do projeto foram aplicados diversos conceitos importantes da **programação reativa**, incluindo:

- APIs reativas utilizando **Spring WebFlux**
- Processamento **assíncrono e não bloqueante**
- Manipulação de streams com **Mono e Flux**
- Comunicação com banco de dados utilizando **ReactiveCrudRepository**
- Integração com **APIs externas de forma reativa**
- Testes automatizados para APIs reativas
- Comparação de desempenho entre **API reativa e API tradicional**

---

# 🌍 Integração com API de Tradução

O projeto também implementa integração com a **API DeepL Translator**, permitindo a tradução automática de descrições de eventos.

Essa funcionalidade é útil para plataformas globais, permitindo que usuários de diferentes países consigam visualizar descrições de eventos no idioma desejado.

A integração foi feita utilizando **WebClient**, mantendo o fluxo da aplicação **totalmente reativo e não bloqueante**.

---

# 🧪 Testes

Para validar o funcionamento da API foram implementados:

- Testes reativos com **WebTestClient**
- Banco de dados **H2 em memória** para testes
- Testes de performance utilizando **Apache AB**

---

# 📈 Performance

Durante os testes de carga, foi possível observar que a **API reativa apresentou melhor desempenho em cenários de alta concorrência**, demonstrando a vantagem do modelo reativo para aplicações que exigem **baixa latência e alta escalabilidade**.
